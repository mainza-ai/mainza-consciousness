## üöÄ **Getting Started**

### Prerequisites
- **Docker** (20.10+) and **Docker Compose** (2.0+)
- **Git** for version control
- **Python 3.11+** for backend development
- **Node.js 18+** for frontend development
- **8GB+ RAM** for development environment

### Development Setup
   ```bash
# Fork and clone the repository
git clone https://github.com/your-username/mainza-consciousness.git
   cd mainza-consciousness

# Create a development branch
git checkout -b feature/your-feature-name

# Start the development environment
./scripts/build-dev.sh

# Verify the system is running
curl http://localhost:8000/health
```

## üìù **Contribution Guidelines**

### Code Style
- **Python**: Follow PEP 8 style guidelines
- **TypeScript**: Use ESLint and Prettier
- **Documentation**: Use clear, concise language
- **Comments**: Include meaningful comments for complex logic
- **Tests**: Write tests for new features

### Git Workflow
1. **Fork** the repository
2. **Create** a feature branch
3. **Make** your changes
4. **Test** your changes thoroughly
5. **Commit** with descriptive messages
6. **Push** to your fork
7. **Create** a pull request

### Commit Messages
Use clear, descriptive commit messages:
```
feat: add consciousness evolution tracking
fix: resolve Neo4j vector index syntax error
docs: update API documentation
test: add consciousness framework tests
```
